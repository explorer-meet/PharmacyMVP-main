02 · Authentication

Authentication & Onboarding

Secure, role-based access for all user types powered by JWT authentication.

🔐
User Registration
Patients & Store Owners
  • Separate registration flows for Patients and Store Owners
  • Email and mobile number validation
  • Password encryption using bcrypt
  • Store Owner profile: Store Name, Owner Name, Licence Number, GST Number
  • Patient profile: Personal details + health information
  • Welcome confirmation on successful registration
✅
Login & Session Management
All Roles
  • JWT token-based authentication
  • Role-based token payload (Patient / Store / Staff)
  • Token persisted in localStorage (medVisionToken)
  • Protected routes redirect unauthenticated users
  • Forgot Password flow
  • Auto logout on token expiry

05 · Prescription

Prescription Management

End-to-end digital prescription workflow — patient uploads, pharmacy review, and approval with a clear status trail.

📤
Patient: Upload Prescription
Digital submission
  • Upload dialog for prescription images
  • Supported formats: PDF, JPG, PNG
  • File size validation before upload
  • AI-powered medicine extraction from uploaded prescription
  • Extracted medicines shown in modal for review
  • Re-upload option for rejected prescriptions
🔍
Store: Review Prescriptions
Approval workflow
  • List of all pending prescriptions from patients
  • Patient details & prescription preview
  • Approve with quote builder (itemised medicines, price, delivery)
  • Reject with documented reason
  • Buttons hidden after action (clean workflow)
Prescription Status Workflow
⏳ Pending
  • Awaiting pharmacy review
  • Patient sees: "Our Pharmacists are carefully reviewing your Prescription"
✅ Approved
  • Prescription verified by pharmacy
  • Quote provided to patient
  • Patient sees: "Your Prescription is Approved. Stay in Touch!!"
❌ Rejected
  • Patient sees rejection reason
  • "Re-upload Prescription" button appears
  • Re-upload resets to Pending for fresh review

07 · Returns

Returns & Refunds

A structured returns workflow with evidence submission, multi-stage review, and refund processing.

↩️
Patient: Raise Return
Self-service return request
  • Raise return request against a delivered order
  • Select items and quantity to return
  • Provide reason for return
  • Upload evidence photos / documents
  • Track return status in My Returns page
⚖️
Store: Process Returns
Review & decide
  • View all open return requests
  • Review evidence submitted by patient
  • Request additional evidence if needed
  • Approve return (triggers refund flow)
  • Reject return with documented reason
  • Mark as Refunded after processing
Return Request Status Flow
Open
  • Request submitted by patient, awaiting store review
Evidence Needed
  • Store requested more evidence from patient
Store Review
  • Evidence submitted, under final review
Approved / Refunded
  • Return approved; refund processing complete
